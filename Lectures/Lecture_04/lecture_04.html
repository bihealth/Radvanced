Creating your own gene sets

Why use your own gene sets?

  • Testing a specific hypothesis
  • Your expert knowledge on the subject is better!
  • Avoiding the pitfall of multiple testing

What do we need: tmod

  • A mapping between genes and gene sets

We need a named list. Names on that list should be identifiers of the gene sets (preferably short). The list should contain character vectors. The elements of these vectors should be the gene identifiers.

  • A data frame describing the gene sets

tmod requires in addition a data frame which is the annotation of the gene sets. It must contain the columns ID and Title, but can also contain other columns which are then used for subsetting.

Getting the mapping

Say, we have a couple of gene sets saved in an XLSX file, "custom_gs.xlsx" with multiple sheets.

Getting the mapping

library(readxl)
excel_sheets("custom_gs.xlsx")
## [1] "Jak_Stat"      "Interferon"    "MycSignalling"
cgs <- lapply(1:3, function(i) read_xlsx("custom_gs.xlsx", sheet=i))

Getting the mapping

This list contains tibbles (a sort of a data frame) instead of character vectors. To get what we need we need only the first column of each spreadsheet!

cgs <- lapply(1:3, 
  function(i) read_xlsx("custom_gs.xlsx", sheet=i)[[1]])
names(cgs) <- excel_sheets("custom_gs.xlsx")

Creating tmod database

tmod uses objects which contain both the data frame and the list. cluster profiler simply uses the mapping (but its output is less informative).

gs.df <- data.frame(ID=names(cgs), 
  Title=c("JAK / STAT pathway signalling", 
          "Interferon a+g signalling",
          "MYC targets"))
tcgs <- makeTmod(modules=gs.df, modules2genes=cgs)
tcgs
## An object of class "tmod"
##  3 modules, 687 genes

Creating gene sets from MSigDB

How to create the cgs/tcgs from MSigDB.

h.names <- unique(h.df$gs_name)
h.cgs <- lapply(h.names, function(n) 
  h.df$human_gene_symbol[ h.df$gs_name == n ])
h.names <- gsub("HALLMARK_", "", h.names)
names(h.cgs) <- h.names

htmod <- makeTmod(modules=data.frame(ID=h.names, Title=h.names), modules2genes=h.cgs)
htmod
## An object of class "tmod"
##  50 modules, 4383 genes

For cluster profiler

Cluster profiler uses in its functions the option TERM2GENE which takes a data frame with gene set names in one column and gene ids in the other.

It is really easy to convert a gene set list to a gene set data frame.

Of course, we can use the results directly.

cgs.df <- Reduce(rbind,
  lapply(names(cgs), function(n) data.frame(ID=n, GENE=cgs[[n]])))

More on tmod and cluster profiler

Get the newest tmod!

Github is a development platform, extremely convenient for tracking the versions of your project (not only for programming!).

devtools is a great R package for developing packages. devtools::install_github allows installation of R packages directly from github.

install.packages("devtools")
devtools::install_github("https://github.com/january3/tmod")

Using cluster profiler HG

Input:

  • gene: foreground set
  • universe: all genes
  • TERM2GENE: data frame mapping gene sets to genes
sign  <- !is.na(res$padj) & res$padj < .01 & abs(res$log2FoldChange) > 1
fg <- res$SYMBOL[sign]
res.cphg <- enricher(fg, universe=res$SYMBOL, TERM2GENE=cgs.df)
head(res.cphg)[,1:5]
